## อธิบายการทำงาน

## 1. การวางโครงสร้างแม่ (Base Class: Product)
เปรียบเสมือน "แบบแปลนกลาง" ที่สินค้าทุกชิ้นในร้านต้องมีเหมือนกัน

Encapsulation (การซ่อนข้อมูล): เราใช้ _price เพื่อไม่ให้คนอื่นเข้ามาแก้ไขราคาได้โดยตรงจากภายนอก

Getter/Setter: * Getter (get price): ทำหน้าที่ส่งค่าราคาออกไปให้คนอื่นดู

Setter (set price): เป็นจุดคัดกรอง (Validation) ก่อนจะบันทึกค่า ถ้าใส่ราคาติดลบ ระบบจะปฏิเสธและแจ้ง Error ทันที ช่วยป้องกันความผิดพลาดของข้อมูล (Data Integrity)

## 2. การสืบทอด (Inheritance)
เราสร้าง Class ลูกขึ้นมาเพื่อขยายความสามารถจาก Class แม่ โดยไม่ต้องเขียนโค้ดซ้ำ

StructuralMaterial (วัสดุโครงสร้าง): รับมรดก (id, name, price) มาจาก Product แต่เพิ่มคุณสมบัติ weightKg (น้ำหนัก) เข้าไปเอง

PowerTool (เครื่องมือไฟฟ้า): รับมรดกมาเหมือนกัน แต่เพิ่ม warrantyMonths (ระยะประกัน) และมีฟังก์ชัน testTool() ที่เอาไว้เช็คสภาพเครื่องมือโดยเฉพาะ

คำสั่ง super(): คือการส่งค่าที่รับมาใน Constructor ของลูก กลับไปให้พ่อแม่จัดการเก็บข้อมูลพื้นฐานให้

## 3. การเขียนทับ (Method Overriding)
แม้ว่าทั้งวัสดุก่อสร้างและเครื่องมือจะมีคำสั่ง displayDetails() เหมือนกัน แต่พฤติกรรมการแสดงผลต่างกัน

ใน Class ลูกทั้งสอง เราใช้ @override เพื่อเขียนการแสดงผลใหม่ให้เหมาะสมกับประเภทสินค้านั้นๆ เช่น วัสดุหนักจะโชว์น้ำหนัก ส่วนเครื่องมือจะโชว์ระยะเวลาประกัน

นี่คือส่วนที่ทรงพลังที่สุดใน main():
